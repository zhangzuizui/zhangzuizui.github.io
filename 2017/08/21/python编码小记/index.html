python编码小记

发表于 2017-08-21 | 分类于 数据处理
被python2和python3的编码(主要是python2)整的欲仙欲死的我今天总算是解放了

最大的不同-python2与python3字符串的默认存储方式

python3

python3的编码就十分舒心了

如图可以发现python3中仿佛去掉了u’’这种东西，输入的u’\u4e2d\u6587’，u’中文’和’中文’是同一个东西，并以Unicode字符的形式存放在变量里，len(s)代表的是字符串的长度，若需要得到字符串的编码，则需要对其encoding，编码后得到的bytes类型数据表示字符串占的字节数

python2

python2就有话要说了

可以发现对于’中文’，python2它直接就以utf8编码后的字节码？形式存放了，加u后，才是以Unicode的形式存放。并且没有bytes这个类型，而是专门的有一个Unicode类。

小结

综上，python2和python3在编码上最大的不同应该是在默认的存储方式上，并且相比之下python2的编码逻辑略显混乱，其实只需要注意到默认存储方式不同这一点，就能很好的处理python2的编码了
